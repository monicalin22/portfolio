The objective of this brief project was to create a more consolidated experience for users of Microsoft Word. Though Word is incredibly useful when it comes to drafting documents, it has no built-in function to open tabs in place of new windows, often resulting in cluttered screens. To solve this user experience issue, I added a feature that allows documents to be opened as tabs.

Understanding the Problem

As college student, I write a lot of papers. I’ve always had difficulty organizing and finding documents, though, because multiple Word files were opened in different windows. I complained to my roommate about this, and she felt similarly, agreeing that tabs as an added feature would improve our user experience. Surely, my roommate and I were not alone.

So, to gauge how others felt about this idea, I conducted general research and interviewed 6 people to get their opinions regarding Microsoft Word. I spoke to 3 fellow classmates, 1 teaching assistant, 1 professor, and also 1 employee at a tech startup.

The main takeaways that I discovered were:‍‍

  • ‍Everyone uses Microsoft Word, and they’ve all been familiar with it for as long as they’ve known. They have been alongside Word for much of its UI/UX transformations.
  • The UI/UX of the program is mostly intuitive, and most interviewees said the application has improved greatly over the years.
  • ‍Comments are difficult to add, and all my interviewees used this feature at least sometimes.
  • Word’s main competitor is Google Docs. Interviewees liked Google Docs because of the easy sharing capability, efficient commenting system, and the ability to keep multiple documents in the same window. Some students, though, thought that the UI/UX and advanced features were lacking.
  • I asked users if they felt that they had too many Word documents open at times, and all of them stated that whether they were writing essays, copy, or any other written content, sometimes, they had to reference other documents, making their screens busy and difficult to navigate.
  • Finally, when asked about what features they would implement to improve Word, they mentioned an improved commenting system or adding tabs to help declutter digital workspaces. (For my UI/UX design project, I will be implementing the latter, which is adding tabs as a feature in Word.)

  • ‍Google Docs: Most similar to Word. Online so allows for easy sharing, commenting, and collaboration. Seems easier to use than Word at first glance. Documents can be opened in the same web browser or different ones.
  • Pages: Most similar to Google Docs and Word. Not as complex as either. Offers tabs in the same window. Just usable enough features to get by, but feels rather simplistic.
  • ‍Notes on MacBook: Rather simple. For projects that are not complex. Quick and convenient. Organized by notes as tabs and folders.
  • Evernote: Very similar to “Notes,” but a bit more complex. Notes organized by folders, favorites, and tabs. All in the same browser.
  • Adobe Acrobat Pro: Not a word processing application, but allows users to read documents. Has tabs and editing tools. Organized mainly by different features. Quite complex.
  • Office Tabs: A feature that adds tabs to all Word products, but does not inherently come with Microsoft Word. Must be downloaded outside of the application. Not a well-known feature. Hard to find unless you’re specifically looking for it. (Will not be graphed.)

Usability Testing

After finishing up my wireframe and prototype, I tested the usability of my new added feature with 5 students, 3 of whom I originally interviewed and 2 close friends.

From this activity, I learned that:

  • Users really liked this idea and could clearly see the utility in it. They asked why Microsoft Words didn’t have a feature to add tabs already.
  • Users thought that my design was intuitive, clear, and in line with how the feature looks on other apps. Locating where the tab feature is was simple as they just moved towards the usual button that opened new documents.
  • ‍Notes on MacBook: Rather simple. For projects that are not complex. Quick and convenient. Organized by notes as tabs and folders.
  • Users felt that this feature would help them keep their documents organized. The main purpose of my project was to make sure that users felt less cluttered and more efficient when they used Microsoft Word. All interviewees displayed great enthusiasm that my feature would help them feel exactly that.
  • Users wondered what the screen would look like if there were more tabs than could fit on the blue bar. Would they stack or would it appear similar to how Google Chrome organizes its tabs? (Personally, I would design something that is similar to the latter.)
  • Users wondered if there was a feature to open files as tabs in the same window as well. Though this was not part of my original wireframe and prototype, it is an important feature that I accounted for in one of my storyboard scenarios.

The Final Product & Implemented Solution

After conducting my usability test, I put together my final prototype. You can see an animation of it below! Click anywhere on the visual to be taken to the Figma prototype so you can interact with it yourself.

To access “New Tab,” simply go to File > New Document > New Tab. Under “New Document,” users can also find the “New Window” option if that’s what they’d prefer.

‍(A feature such as opening files as tabs instead of just new files would look similar. A user should click File > Open/Open Recent to select the files they wish to upload. Afterwards, a message should pop-up asking the user if they want these files opened in multiple windows or as multiple tabs in one window.)
